# Macro Flow / 宏观金融流动学习器

一个面向视觉型学习者的个人宏观金融关系学习小应用（V1 原型）。

## 本地运行

```bash
python3 -m http.server 4173
```

然后访问 <http://localhost:4173>。

## 如果你在 GitHub Codespaces 打不开页面

很多时候不是项目问题，而是 **端口面板没显示** 或 **服务没启动**。按下面最短流程走：

### 1) 先确认服务是否真的启动

在终端执行：

```bash
python3 -m http.server 4173
```

看到类似 `Serving HTTP on 0.0.0.0 port 4173` 才算启动成功。

### 2) 打开 Ports 面板（你看不到 Ports 时）

在 Codespaces 网页版 VS Code 顶部菜单：

- `View` → `Terminal`（先确保终端可见）
- `View` → `Ports`（如果有这个项，直接点）

如果菜单里没有 `Ports`：

- 按 `F1` / `Ctrl+Shift+P` 打开命令面板
- 输入并执行：`Ports: Focus on Ports View`

### 3) 手动添加端口 4173

在 Ports 面板里点击 `Forward a Port`，输入 `4173`。

添加后会出现一行端口记录，通常会有一个可点击的 URL（或右键 `Open in Browser`）。

### 4) 如果端口还是不出现

用这几个命令排查：

```bash
# 看 4173 是否在监听
ss -ltnp | rg 4173

# 看本容器内是否可访问
curl -I http://127.0.0.1:4173
```

- 如果 `curl` 返回 `200`，说明应用是好的，问题在 Codespaces 端口面板/转发。
- 如果 `curl` 不通，说明服务没起来，回到第 1 步。

### 5) 最常见“看不到 Ports”的原因

- 没有启动任何监听端口的服务。
- 窗口布局把 Ports 面板隐藏了。
- 你开的是仓库页面，不是 Codespace 编辑器页面。
- 浏览器插件拦截弹窗/新标签导致你以为没打开预览。
